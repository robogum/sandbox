## Google Custom Search API

Google Corabから新しいノートブックを作成する。
ノートブックで選択できる「モデル」は、大本とは無関係なんですね。proを使うためには、「upgurade」しないといけないみたいです。


『日本語で、Google Custom Search APIを使って検索するスクリプトを作成してください。』


コードは、サクッと作成してくれました。あとは、必要な「GOOGLE_API_KEY」と「GOOGLE_CSE_ID」を持ってくる必要がります。

『GOOGLE_API_KEYは、どこから取得しますか。』

わからない所があれば、聞けば教えてくれますが、この辺から、敷居が一段上がる感じですね、Google Cloud に登録が必要になります。
 

コードを実行した結果

```
{'code': 403, 'message': 'This project does not have the access to Custom Search JSON API.', 'errors': [{'message': 'This project does not have the access to Custom Search JSON API.', 'domain': 'global', 'reason': 'forbidden'}], 'status': 'PERMISSION_DENIED'}
```


このエラーで、悩む事2時間。 cursorで確認してみたら、

```
公式ドキュメントには、次の注意書きがあります。 Custom Search JSON API は新規顧客には提供終了しています。 既存顧客は 2027年1月1日まで利用可能。 つまり、Console で API を有効にしても、新しい Google Cloud プロジェクトでは利用権限が付与されません。 今回のエラー： This project does not have the access to Custom Search JSON API. は、API キーや cx の設定ミスではなく、新規プロジェクトにはそもそもアクセス権がない ことを示している可能性が高いです。
```

この結果を見た後で、Geminiに問い合わせをしたら

```
Google Cloudコンソール上でAPIを「有効」に設定し、課金（Billing）を有効にして、正しいAPIキーや検索エンジンID（cx）を設定していたとしても、新しく作成されたプロジェクトからは一律でこの 403 Permission Denied エラーが返される仕様に変更されました。
```

ということなので、あきらめました。 今回のようにどこも間違っていないのに、ちゃんと実行できない事もあります。（AIだけじゃないですけど） 他で確認する事も必要です。
